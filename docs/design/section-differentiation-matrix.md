# Section Differentiation Matrix

## Purpose
Prevent Aura, Scent, Style, Sound, and Self from converging into the same template with relabeled headers.

## Core Rule
Each section must have a distinct interaction grammar, visual rhythm, and decision output.

## Matrix
| Section | User Intent | Signature Interaction | Visual Rhythm | Output Shape | Failure If Generic |
|---|---|---|---|---|---|
| Aura | Define moment-level presence | Cinematic generation -> ceremonial reveal | Hero-first, staged reveal | One integrated aura direction | Looks like a generic chat result panel |
| Scent | Decide from owned wardrobe | Collection-first selection + situation refinement | Dense shelf + detail sheet contrast | Decisive “reach for” recommendations | Feels like random fragrance suggestions |
| Style | Translate aura to wearable composition | Silhouette/palette assembly workflow | Structured composition blocks | Outfit direction with hierarchy | Becomes another text recommendation tab |
| Sound | Set environmental frequency | Context-to-sound scene builder | Tempo/energy pacing, layered cues | Curated sonic scene with mood progression | Reads like playlist listicle output |
| Self | Reflect patterns and identity arc | Timeline/reflection checkpoints | Quiet, archival, slower cadence | Insight summaries + personal trajectory | Feels like settings/account page |

## Differentiation Guardrails
- Aura must own ceremony and reveal pacing.
- Scent must anchor to wardrobe reality before advice.
- Style must show composition logic, not only descriptive prose.
- Sound must express progression (build, sustain, release), not static picks.
- Self must prioritize continuity over novelty.

## Shared Elements (Allowed)
- Global typography system.
- Brand color tokens.
- Consistent bottom navigation behavior.
- Reusable primitives (sheet, card, button) with section-specific composition.

## Shared Elements (Not Allowed)
- Same hero layout across all sections.
- Same CTA language pattern across all sections.
- Same animation sequence for every section entry.
- Same empty-state structure and copy in all sections.

## Review Checklist
- Can screenshots of each section be distinguished with headers hidden?
- Does each section produce a different “type” of decision?
- Does each section have at least one unique interaction not reused verbatim elsewhere?
- Does motion behavior reinforce section identity instead of repeating defaults?
